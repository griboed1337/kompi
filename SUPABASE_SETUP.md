# Настройка Supabase для PC Configurator

## 1. Создание проекта в Supabase

1. Перейдите на [supabase.com](https://supabase.com)
2. Создайте новый проект
3. Дождитесь завершения настройки

## 2. Настройка базы данных

1. В панели управления Supabase перейдите в **SQL Editor**
2. Скопируйте и выполните содержимое файла `schema.sql`

## 3. Настройка аутентификации (опционально)

Если планируете использовать аутентификацию:

1. Перейдите в **Authentication** > **Settings**
2. Настройте провайдеры (Google, GitHub и т.д.)
3. В **Authentication** > **Users** создайте тестового пользователя

## 4. Получение API ключей

1. Перейдите в **Settings** > **API**
2. Скопируйте **Project URL** и **anon/public key**
3. Добавьте их в файл `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=your_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
```

## 5. Настройка RLS (Row Level Security)

Если используете RLS политики:

1. В **Authentication** > **Settings** включите RLS
2. Или настройте политики в SQL Editor

## 6. Тестирование подключения

1. Запустите приложение: `npm run dev`
2. Проверьте API эндпоинты:
   - `http://localhost:3000/api/products` - получение продуктов
   - `http://localhost:3000/api/scrape` - запуск скрейпинга

## 7. Мониторинг и логи

- В **Table Editor** просматривайте данные в таблицах
- В **Logs** мониторьте запросы к API
- В **Storage** (если нужно) настройте хранение изображений

## 8. Производство

Для продакшена:

1. Настройте кастомный домен в Supabase
2. Обновите переменные окружения
3. Настройте резервное копирование в **Settings** > **Database**

## Поддержка

Если возникнут проблемы:
- Проверьте логи в Supabase Dashboard
- Убедитесь в правильности API ключей
- Проверьте RLS политики
